# Ansible Configuration Management

This directory contains the Ansible configuration used to automate server provisioning, application deployment, and environment configuration for the TaskApp DevOps CI/CD Platform.

Ansible is responsible for configuring infrastructure provisioned by Terraform and ensuring application services are deployed consistently across environments.

---

## Overview

This Ansible implementation automates:

- Base server configuration
- Frontend application deployment
- Backend application deployment
- Nginx configuration
- Environment variable management
- Systemd service management
- PostgreSQL connectivity configuration

The configuration is organized using reusable Ansible roles to promote maintainability and scalability.

---

## Role Architecture

The project uses three primary roles:

### Common Role

Responsible for:

- System updates
- Common package installation
- Application user and group creation
- Directory structure creation
- Shared server configuration

### Frontend Role

Responsible for:

- Node.js installation
- Frontend build deployment
- Nginx installation and configuration
- Static asset hosting

### Backend Role

Responsible for:

- Python installation
- Virtual environment configuration
- Backend application deployment
- Environment variable configuration
- Systemd service creation and management

## Terraform Integration

Ansible consumes infrastructure outputs generated by Terraform.

Terraform outputs are loaded from:

```text
terraform/root/terraform_outputs.json
```

These outputs provide:

- Frontend server information
- Backend server information
- Database endpoint information
- Infrastructure metadata

Terraform infrastructure must be provisioned before running Ansible.

---

## Secrets Management

Sensitive values are stored using Ansible Vault.

Vault file:

```text
inventory/group_vars/backend/vault.yml
```

Examples of protected values:

- Database credentials
- Application secrets
- Environment-specific configuration

Edit encrypted secrets:

```bash
ansible-vault edit inventory/group_vars/backend/vault.yml
```

## Deployment Workflow

Main playbook:

```text
playbooks/site.yml
```

Execution order:

1. Common Role
2. Frontend Role
3. Backend Role

---

## Deployment Commands

### Full Deployment

```bash
cd ansible

ansible-playbook \
  -i inventory/dev.yml \
  playbooks/site.yml \
  --ask-vault-pass
```

### Deploy Frontend Only

```bash
ansible-playbook \
  -i inventory/dev.yml \
  playbooks/site.yml \
  --limit frontend \
  --ask-vault-pass
```

### Deploy Backend Only

```bash
ansible-playbook \
  -i inventory/dev.yml \
  playbooks/site.yml \
  --limit backend \
  --ask-vault-pass
```

### Dry Run

```bash
ansible-playbook \
  -i inventory/dev.yml \
  playbooks/site.yml \
  --check \
  --diff \
  --ask-vault-pass
```

## Verification

### Verify Frontend

```bash
sudo systemctl status nginx
curl -I http://localhost
```

### Verify Backend

```bash
sudo systemctl status taskapp-backend
sudo journalctl -u taskapp-backend -n 100
```

### Verify Connectivity

```bash
psql -h <RDS_ENDPOINT> \
     -U taskapp_user \
     -d taskapp \
     -p 5432
```

## Common Operations

### List Inventory

```bash
ansible-inventory -i inventory/dev.yml --list
```

### Ping Hosts

```bash
ansible -i inventory/dev.yml all -m ping
```

```bash
ansible-playbook \
  -i inventory/dev.yml \
  playbooks/site.yml \
  -vv \
  --ask-vault-pass
```
